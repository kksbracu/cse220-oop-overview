OOP overview in python

1. Classes and Objects

A class is a blueprint for creating objects. Objects are instances of a class. A class can contain attributes (variables) and methods (functions) that define the behavior of the objects created from it.

Example: 

class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    def start(self):
        print(f"The {self.make} {self.model} is starting.")

# Creating an object (instance) of the Car class
my_car = Car("Toyota", "Camry", 2021)
my_car.start()  # Calling the method on the object

Object: An instance of a class, created with specific values.

another_car = Car("Honda", "Civic", 2020)

2. Attributes and Methods

  • Attributes: Variables associated with an object that store the data of the object.
  • Methods: Functions associated with an object that define behaviors.

Example: 

class Dog:
    def __init__(self, name, breed):
        self.name = name  # Attribute
        self.breed = breed  # Attribute

    def bark(self):  # Method
        print(f"{self.name} says Woof!")

# Using the class
dog1 = Dog("Buddy", "Golden Retriever")
dog1.bark()  # Calling the method

3. The __init__ Method (Constructor) {#3-the-init-method-constructor }

  • The __init__ method initializes an object when it's created. It’s automatically called when you create a new object.

Example: 

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

person1 = Person("Alice", 30)  # Calls __init__ to set name and age

4. The self Keyword

  • self refers to the instance of the class. It is used to access the attributes and methods of the current object from within the class.

Example: 

class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

    def describe(self):
        print(f"Book: {self.title} by {self.author}")

book1 = Book("1984", "George Orwell")
book1.describe()  # 'self' allows access to instance variables

5. Encapsulation

  • Encapsulation means hiding the internal details of an object and restricting access to them. You can make attributes private by prefixing them with __ (double underscore), which makes them inaccessible from outside the class.
  • It also provides getter and setter methods to safely access and modify private attributes.

Example: 

class Account:
    def __init__(self, owner, balance=0):
        self.__owner = owner  # Private attribute
        self.__balance = balance  # Private attribute

    def get_balance(self):
        return self.__balance

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
        else:
            print("Deposit amount must be positive.")

account1 = Account("John", 1000)
print(account1.get_balance())  # Accessing balance through a method
account1.deposit(500)
print(account1.get_balance())

6. Inheritance

  • Inheritance allows a class to inherit attributes and methods from another class. This enables code reuse and extension of existing functionality.

Example: 

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name} makes a sound")

class Dog(Animal):  # Dog inherits from Animal
    def speak(self):
        print(f"{self.name} barks")

dog = Dog("Buddy")
dog.speak()  # Overridden method of Dog class

7. Polymorphism

  • Polymorphism allows objects of different classes to be treated as objects of a common superclass. The method that gets called depends on the type of the object.

Example: 

class Cat(Animal):
    def speak(self):
        print(f"{self.name} meows")

animals = [Dog("Rex"), Cat("Whiskers")]

for animal in animals:
    animal.speak()  # Polymorphism: calling the appropriate speak method

8. Abstraction

  • Abstraction means hiding complex implementation details and showing only the necessary functionality. This is usually achieved using abstract classes and methods, but you can also abstract functionality by defining high-level methods in the class.

Example: 

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

circle = Circle(5)
print(circle.area())  # The area method is defined in the subclass

9. Class vs. Instance Methods

  • Class methods are methods that are bound to the class, not instances. They are defined using @classmethod and take cls as the first parameter.

Example: 

class Car:
    count = 0  # Class variable

    def __init__(self, make, model):
        self.make = make
        self.model = model
        Car.count += 1  # Modifies class variable

    @classmethod
    def get_car_count(cls):
        return cls.count

car1 = Car("Toyota", "Corolla")
car2 = Car("Honda", "Civic")
print(Car.get_car_count())  # Class method accessing the class variable
  • Instance methods are bound to the instance and can access instance-specific data.
car1 = Car("Toyota", "Corolla")
car2 = Car("Honda", "Civic")
print(car1.get_car_count())  # Calling class method on instance

10. Example: Book and Author Objects

  • In this example, a Book object will have an Author object as an attribute. The Author object will contain details about the author, and the Book object will contain details about the book and a reference to the Author.

  • Example:

class Author:
    def __init__(self, name, birth_year):
        self.name = name
        self.birth_year = birth_year

    def get_author_info(self):
        return f"Author: {self.name}, Born: {self.birth_year}"

class Book:
    def __init__(self, title, year, author):
        self.title = title
        self.year = year
        self.author = author  # The Book object has an Author object

    def get_book_info(self):
        author_info = self.author.get_author_info()  # Accessing Author's method
        return f"Book: {self.title}, Year: {self.year}, {author_info}"

# Creating an Author object
author1 = Author("J.K. Rowling", 1965)

# Creating a Book object with an Author object
book1 = Book("Harry Potter and the Sorcerer's Stone", 1997, author1)

# Getting information about the Book and its Author
print(book1.get_book_info())
  • Output
Book: Harry Potter and the Sorcerer's Stone, Year: 1997, Author: J.K. Rowling, Born: 1965
  • Explanation:
  • Author Class: The Author class has attributes like name and birth_year. It also has a method get_author_info() to return information about the author.
  • Book Class: The Book class has attributes like title, year, and a reference to an Author object. The get_book_info() method combines information about the book and its author.
  • Connecting Objects: The Book object is connected to the Author object by passing the author object as an argument to the Book constructor.

5. Encapsulation

  • Encapsulation means hiding the internal details of an object and restricting access. Attributes are made private, and getter and setter methods are used to access them.

Example: 

class Account {
    private String owner;
    private double balance;

    Account(String owner, double balance) {
        this.owner = owner;
        this.balance = balance;
    }

    // Getter method
    public double getBalance() {
        return balance;

    // Setter method

        if (amount > 0) {

        } else {
            System.out.println("Deposit amount must be positive.");

    }
}

public class Main {

        System.out.println(account1.getBalance());
    }
}

6. Inheritance

  • Inheritance allows a class to inherit attributes and methods from another class, enabling code reuse.

class Animal {

  • Abstraction hides implementation details while exposing only essential features.

9. Static vs Instance Methods

  • Static methods belong to the class and do not require an object.
  • Instance methods require an object of the class.

Example: 

class Car {
    static int count = 0;

    Car() {
        count++;
    }

    static int getCarCount() {
        return count;
    }
}

public class Main {
    public static void main(String[] args) {
        Car car1 = new Car();
        Car car2 = new Car();
        System.out.println(Car.getCarCount()); // Static method called on the class
    }
}

10. Example: Book and Author Objects

  • A Book object will have an Author object as an attribute.

Example: 

class Author {
    String name;
    int birthYear;

    Author(String name, int birthYear) {
        this.name = name;
        this.birthYear = birthYear;
    }

    String getAuthorInfo() {
        return "Author: " + name + ", Born: " + birthYear;
    }
}

class Book {
    String title;
    int year;
    Author author;

    Book(String title, int year, Author author) {
        this.title = title;
        this.year = year;
        this.author = author;
    }

    String getBookInfo() {
        return "Book: " + title + ", Year: " + year + ", " + author.getAuthorInfo();
    }
}

public class Main {
    public static void main(String[] args) {
        // Creating an Author object
        Author author1 = new Author("J.K. Rowling", 1965);

        // Creating a Book object with an Author object
        Book book1 = new Book("Harry Potter and the Sorcerer's Stone", 1997, author1);

        // Getting information about the Book and its Author
        System.out.println(book1.getBookInfo());
    }
}
  • Output
Book: Harry Potter and the Sorcerer's Stone, Year: 1997, Author: J.K. Rowling, Born: 1965

  • Explanation

  • Author Class:

    • The Author class has attributes like name and birthYear.
    • It also has a method getAuthorInfo() that returns a formatted string with the author's details.

  • Book Class:

    • The Book class has attributes like title, year, and an Author object as an attribute.
    • The method getBookInfo() retrieves the book details and calls getAuthorInfo() from the Author class to include the author's information.

  • Connecting Objects:

    • The Book object is connected to an Author object by passing the author1 instance to the Book constructor.
    • This demonstrates composition, where an object (Book) contains another object (Author) as an attribute.

This example showcases Encapsulation (private attributes via constructor parameters) and Composition (Book contains Author) in Object-Oriented Programming (OOP) with Java.
